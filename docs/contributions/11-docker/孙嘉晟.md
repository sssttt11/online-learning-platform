### 1 前端 Dockerfile 编写与优化 (Frontend Containerization)

负责基于现代前端构建工具（如 Vite / Webpack）及静态资源服务器（如 Nginx）的镜像构建。

- **多阶段构建 (Multi-stage Build)**：分离“构建环境”与“运行环境”。在构建阶段使用完整的 Node.js 环境生成静态文件；在运行阶段仅保留轻量级的 Nginx 镜像，大幅度缩减生产镜像体积。
- **缓存层优化 (Layer Caching)**：合理利用 Docker 缓存机制。优先复制 `package.json` 和包管理锁文件（如 `package-lock.json` 或 `pnpm-lock.yaml`）进行依赖安装（`npm install`），避免因业务代码修改导致依赖重复下载。
- **Nginx 配置嵌入**：编写并打包生产环境的 `nginx.conf`，处理单页应用（SPA）的路由刷新 404 问题，并开启 Gzip 压缩等性能优化。

### 2 后端 Dockerfile 编写与优化 (Backend Containerization)

负责后端服务的容器化封装，确保服务在云端安全、稳定、高效地运行。

- **精简基础镜像**：生产环境严禁使用全量操作系统镜像，统一采用 `alpine` 或 `slim` 等精简版基础镜像，降低安全漏洞风险并减少磁盘占用。
- **多阶段编译**：针对编译型或需要打包的后端语言，在第一阶段进行编译/打包，第二阶段仅提取最终的运行产物（如可执行文件、Jar 包或打包后的 Node 服务），保持运行环境的纯净。
- **生产环境配置分离**：确保 Dockerfile 具备良好的环境适应性，通过 `ENV` 或 `EXPOSE` 明确对外暴露的参数与端口，配合容器启动时的环境变量注入。
