## 遇到的问题和解决
1. 问题：AI 建议使用 DOMPurify 净化富文本，但项目内引入该库后导致 Vue 的响应式绑定出现了兼容性警告。
 解决：最终改为在后端进行内容过滤（更安全），前端通过 Vue 的文本插值自动转义来防御基础 XSS，保持了代码简洁。
## 心得体会在前端安全开发中，最大的挑战在于“防御边界”。我意识到，仅仅在 UI 上隐藏按钮（Security by Obscurity）是无效的。真正的安全在于前端的优雅降级与后端的严苛校验相结合。在 Vibe Coding 的过程中，我们利用 AI 快速生成功能代码，但必须时刻保持对 innerHTML、localStorage 和 API 异常处理的警惕，将安全审查作为开发流中的固定环节，才能在效率与安全性之间取得平衡。
