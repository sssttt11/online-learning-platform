# 在线课程平台前端架构设计文档

## 一、技术栈与核心规范

### 1.1 技术栈选型

基于图片中规范，结合在线课程平台业务特性，确定核心技术栈：

|   层级    |          技术选型          |                         说明                         |
| :-------: | :------------------------: | :--------------------------------------------------: |
| 前端框架  |   React 18 + TypeScript    |     组件化开发、强类型约束，降低大型项目维护成本     |
| 样式方案  | Tailwind CSS + CSS Modules |  原子化 CSS 提升开发效率，CSS Modules 解决样式隔离   |
| 状态管理  |       Redux Toolkit        |    全局状态（用户信息、课程列表、购物车）统一管理    |
|   路由    |       React Router 6       |  支持路由懒加载、动态路由（适配课程详情、讲师主页）  |
| 网络请求  |    Axios + React Query     |         统一请求拦截、缓存接口数据、自动重试         |
| UI 组件库 |         Ant Design         |        快速实现表单、表格、弹窗等通用业务组件        |
| 构建工具  |            Vite            |    相比 Webpack，开发环境热更新更快，构建体积更小    |
|   部署    |       Docker + Nginx       | 容器化部署，支持多环境（开发 / 测试 / 生产）快速切换 |

#### 1.2.2 编码规范

1. 组件规范

   - 函数式组件 + Hooks 开发，禁止使用 class 组件；组件命名采用 PascalCase，文件命名与组件名一致。
   - 组件拆分原则：单一职责，通用组件抽离至`src/components/common`，业务组件抽离至`src/components/business`。
   - 组件 props 使用 TypeScript 接口定义，必填参数标注`required`，默认参数使用`defaultProps`。

2. 类型规范

   - 禁止使用`any`类型，所有数据类型必须定义接口（interface）或类型别名（type）。
   - 类型定义按模块分类存放于`src/types/`，避免类型分散定义。

3. 样式规范

   - 优先使用 Tailwind CSS 原子类，复杂样式使用 CSS Modules（文件命名：`[组件名].module.css`）。
   - 禁止内联样式（`style`属性），特殊场景除外。
   - 统一使用 Tailwind 自定义主题（在`tailwind.config.js`中配置主色、辅色、字体等）。

4. API 规范

   - 接口响应统一格式：`{ code: number, data: T, message: string }`，前端统一封装响应拦截器处理。
   - 接口请求参数使用 TypeScript 类型约束，避免任意传参。

5. 禁止事项

   - 禁止直接操作 DOM（如需操作 DOM，使用 React.useRef）。
   - 禁止修改配置文件（除非明确需求，如环境变量、构建配置）。
   - 禁止在组件中直接暴露敏感信息（如接口密钥、用户隐私数据）。

### 2.2 页面组件层级关系

以**课程详情页**为例，展示组件层级：

```
CourseDetailPage（页面组件）
├── Layout（布局组件：Header + Footer + 内容区）
│   ├── CourseHeader（业务组件：课程标题、价格、收藏按钮）
│   ├── CourseTabs（业务组件：标签页切换：介绍/大纲/评价/相关推荐）
│   │   ├── CourseIntro（子组件：课程详情、讲师介绍）
│   │   ├── CourseOutline（子组件：章节列表、视频预览）
│   │   ├── CourseComment（子组件：评论列表、评论输入）
│   │   └── RelatedCourse（子组件：相关课程推荐）
│   └── CourseSidebar（业务组件：购买按钮、课程信息、分享）
└── CommonModal（通用组件：登录弹窗、购买确认弹窗）
```
