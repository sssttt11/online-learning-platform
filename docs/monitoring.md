# 栖学课堂 - 监控与可观测性配置说明

为了保障“栖学课堂”在线上生产环境的稳定运行，本项目已在 Node.js 后端服务中全面引入了基础监控体系。

## 1. 结构化日志管理 (Logging)
- **技术选型**: 采用了业界标准的 `winston` 库。
- **配置详情**: 日志统一采用 JSON 格式输出，记录了 `timestamp`、`level` 以及具体的请求元数据。日志不仅会输出至容器控制台 (Console)，还会持久化存储至 `app.log` 文件中，方便后续集成 ELK 等日志分析系统。

## 2. 探针与健康检查 (Health Check)
- **端点路由**: `/health`
- **实现功能**: 为云平台 (如 Railway) 的负载均衡器提供心跳检测。当请求该接口时，系统会返回标准的 JSON 健康存活回执，包含当前系统时间 `timestamp` 与服务版本 `version`。

## 3. APM 基础指标收集 (Metrics)
我们在 Express 中挂载了全局中间件，用于无侵入式地拦截并统计三大核心指标：
1. **请求计数**: 自动捕获所有进入路由的 HTTP 请求。
2. **响应时间**: 利用 `Date.now()` 计算处理耗时 (`durationMs`)。
3. **错误率区分**: 当 HTTP 状态码 `>= 400` 时，自动触发 `logger.error` 级别进行错误追踪与高亮。
